# FigureDescription schema (shared)

Defines FigureDescription object and per-type content `html-export.md` injects into standalone HTML. Also holds risk rubric `extract.md` uses for `null_field_classification` directives.

## Fields

|Field|Type|Content|
|---|---|---|
|`title`|string|`<matter_name> ŌĆö <category_label>`; fallback `<category_label> ŌĆö <YYYY-MM-DD>`|
|`matter_context`|string|matter_type + jurisdiction (if any) + one-line parties summary|
|`caption`|string|one-line, per-type pattern below|
|`overview`|string|3 sentences: what it shows, why it matters to the matter type, what decision it supports|
|`how_to_read`|string|per-type legend below|
|`observations`|string[]|3-5 plain-language bullets read off the diagram's structure|
|`caveats`|string[]|matter-type base set + diagram-type addendum + date-validity note|

## Per-type caption and legend

`caption` and `how_to_read` content per type (use plain-language name when speaking to user; these strings go in the HTML figure):

|Type|Caption|How to read|
|---|---|---|
|`timeline`|"Timeline: <subject>, <date range>"|Read top to bottom; section headers group events by period.|
|`gantt`|"Gantt chart: <tasks/phases>, <date range>"|Bar width = duration; diamonds = milestones; time runs left to right.|
|`flowchart`|"Flowchart: <process> decision path, <context>"|Follow arrows; diamonds = decisions, rectangles = processes/states.|
|`state diagram`|"State diagram: <process> from <start> to <end states>"|Boxes = states, arrows = transitions, labels = triggers; [*] = entry/exit.|
|`erDiagram`|"Entity relationship diagram: <N> entities, <M> relationships"|Boxes = entities, lines = relationships; cardinality `||` one, `o|` zero-or-one, `}|` one-or-more.|
|`requirementDiagram`|"Requirement diagram: <N> obligations mapped to <M> controls"|Rounded = requirements, square = elements; arrows = satisfies/traces/contains/copies.|
|`sequenceDiagram`|"Sequence diagram: <flow> through <N> actors"|Columns = actors, horizontal arrows = messages; time flows top to bottom.|
|`mindmap`|"Mind map: <root topic>"|Centre = root; branches radiate by level; indentation depth = hierarchy level.|
|`quadrantChart`|"Quadrant chart: <Y label> vs <X label>, <scope>"|Position encodes two scores; top-right = high on both axes.|
|`journey`|"Journey map: <actor> through <N> phases"|Rows = actors, columns = phases; score 1-5 = friction/satisfaction.|

## Risk-classification rubric

Used by `extract.md` `null_field_classification` directives on `obligations[].risk_level`.

- **high**: material breach, termination, indemnification, personal-data exposure, regulatory deadline, fiduciary duty.
- **medium**: payment, notice, reporting, routine covenant, ordinary-course obligation.
- **low**: administrative, informational, best-efforts language without consequence.

## Matter-type caveat sets

- `litigation`: "Does not constitute legal advice. Verify all dates against source documents."
- `compliance`: "Controls listed may not cover all applicable requirements. Engage qualified counsel."
- `corporate`: "Ownership percentages are as of the document date and may have changed."
- `privacy`: "Data flows shown are representative; verify against current processing records."
- all types: "Generated by /legal-diagram on <date>. Accuracy depends on source quality."
