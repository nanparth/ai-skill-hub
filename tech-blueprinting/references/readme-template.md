# README Template

Use this template for public or project-facing README files. Do not add private workspace metadata, or machine-local paths.

## Recommended Sections

1. Title and one-sentence description.
2. Who this is for.
3. Prerequisites.
4. Quick start.
5. Common workflows.
6. Inputs and outputs.
7. Configuration, if any.
8. Troubleshooting.
9. File map.
10. Limitations and safety notes.

## Style Rules

- Start with the actual thing the reader can use.
- Prefer short sentences and concrete examples.
- Say which dependencies are required and which are optional.
- Use neutral paths such as `./docs/`, `./outputs/`, or `<project-root>/...`.
- Do not include personal names, private project names, local drive paths, or hidden private workspace conventions.
- Do not add private workspace metadata fields unless the user explicitly asks for that format.
